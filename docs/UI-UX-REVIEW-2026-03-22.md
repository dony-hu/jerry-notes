# Jerry Notes UI/UX Review 与设计建议

日期：2026-03-22

## 1. 站点当前给人的感觉

第一次来到这个站的人，会先看到一张很有性格的猫鼠横幅，然后进入一个偏严肃、偏工具化的知识空间。

这个组合并不是错的，但现在它还没有被设计成一种“有意识的反差”，所以体验像是两套气质同时在说话：

- 上半部分在说：这是一个有个性的个人站
- 中间内容区在说：这是一个内部知识库 / 工作台
- 文章详情页的导出、公众号稿、3条摘要又在说：这是一个内容生产工具

结果就是，品牌气质、信息层级、操作意图都还没有完全收拢到同一个故事里。

## 2. UI Review 结论

### 2.1 最值得优先解决的问题

#### 问题 1：站点身份不够清晰，个人站与内部知识库的心智在打架

当前首页把“个人表达”“文章索引”“内部访问控制”“内容生产工具”都放进了同一个第一屏里。

这会带来一个典型问题：

- 未登录用户不确定这个站主要是公开阅读，还是一个需要授权的内部系统
- 已登录用户也没有明显感受到自己进入了“内部工作视图”

站点不是不能同时承载这两种角色，但需要更明确的模式感。

#### 问题 2：首页的“浏览决策成本”偏高

列表区现在主要依靠：

- 标题
- 日期
- internal / public 标记
- 标签

对熟悉内容的人够用，但对“快速扫一眼决定点哪篇”的用户不够友好。缺少摘要、内容分组、主次层级之后，首页更像“原始索引”，不太像“可浏览的知识首页”。

#### 问题 3：头部区域的控件都在争夺注意力

当前头部同时放了：

- 横幅
- 标题与副标题
- 日夜主题切换
- 飞书登录 / 用户卡片
- 文章 / 关于导航

这些元素都合理，但现在的组织更像“功能并排摆放”，还不像“有明确主次的头部系统”。

#### 问题 4：阅读页操作栏过于平均，缺少主次

详情页顶部同时露出：

- 返回列表
- 导出 PDF
- 公众号稿
- 3条摘要

这几个动作的重要性对所有用户并不一致，但现在它们在视觉上几乎同级。阅读动作和生产动作没有被分层，会让页面看起来更像工具面板，而不是内容阅读器。

#### 问题 5：筛选区是“有用的”，但不是“轻的”

右侧索引筛选确实能帮助查找，但它现在还是一块独立的小控制台：

- 默认一直存在
- 和内容区视觉绑定不够紧
- 在移动端会上移，容易打断用户先看内容的节奏

更理想的状态应该是：筛选能力仍然在，但默认更轻、更像辅助层。

## 3. 设计北极星

我建议这个站的设计方向收敛成一句话：

**“有温度的个人知识站，带着轻量内部工作台能力。”**

这句话的含义是：

- 第一身份仍然是内容站，不是管理后台
- 登录是为了“解锁更深层内容”，不是把整站变成企业系统
- 内部内容的感受应该是“更专业、更有秩序”，而不是“更厚重、更拥挤”

## 4. 推荐的体验重构方向

### 4.1 信息架构：从“单列索引”改成“模式明确的知识首页”

首页建议拆成三个层次：

1. 顶部身份层
- 只负责站点身份、登录状态、主题切换

2. 内容入口层
- 明确告诉用户当前看的是哪一类内容
- 例如：`全部` / `外部` / `内部`

3. 内容浏览层
- 列表卡片 + 辅助筛选

这样用户在前 5 秒内就能回答三个问题：

- 这是什么站
- 我当前看到的是哪类内容
- 我接下来应该往哪里点

### 4.2 首页结构建议

推荐改成下面这个节奏：

```text
[Utility Bar]
Jerry Notes | 主题切换 | 登录 / 用户状态

[Hero]
站点标题
一句更明确的副标题
一句辅助说明：公开文章 + 登录后可见内部知识

[Mode Tabs]
全部 / 外部 / 内部

[Primary Feed]
按时间展示内容卡片

[Secondary Controls]
标签筛选 / 月份筛选 / 重置
```

核心变化不是增加很多东西，而是让“模式”和“内容”先于“工具”。

### 4.3 登录体验建议

现在登录已经可用，但体验上可以更“像一个邀请”，而不是“一个系统按钮”。

建议：

- 未登录时：
  - 登录按钮放在 utility bar 或 hero 旁边
  - 配一句极短说明：`登录后查看内部文章与工作手册`

- 已登录时：
  - 使用更轻的头像状态 pill
  - 不要像一个大卡片压在标题旁边
  - 视觉目标是“已登录但不抢戏”

### 4.4 文章列表建议

列表应该从“索引”升级成“可浏览卡片”。

每个列表项建议包含：

- 标题
- 一行摘要
- 日期
- 标签
- 可见性标记

其中“内部 / 外部”不只是 metadata，而要成为用户的预期管理：

- `外部`：冷色、轻量、开放
- `内部`：暖色、轻量锁定感、更克制

如果以后登录后允许看内部内容，首页还可以直接把内部内容聚成一个区块，而不是完全混排。

### 4.5 阅读页建议

阅读页现在内容本身已经比之前稳定很多，但上方操作区还偏“工具栏”。

建议把动作拆成两层：

- 一级动作：
  - 返回列表
- 二级动作：
  - 导出 PDF
  - 公众号稿
  - 3条摘要

二级动作可以做成：

- `更多操作` 下拉
- 或右上角轻量图标组

这样阅读页会更像“先读，再操作”。

### 4.6 关于区建议

`关于我` 现在放在首页底部没有问题，但文案和页面成熟度不匹配。

页面已经不再是“第一版小实验”了，它已经具备：

- 权限控制
- 双内容层
- 导出能力
- slides 能力

所以这里建议改成更成熟的一句话：

- 这是什么站
- 公开与内部内容分别是什么
- 这个站的写作 / 记录范围是什么

## 5. 视觉设计建议

### 5.1 视觉气质

我不建议把它改成标准企业后台，也不建议继续停留在“个人博客 + 工具按钮拼装”的状态。

更适合它的是：

- **内容型界面** 的呼吸感
- **知识工具** 的秩序感
- **个人站** 的一点辨识度

### 5.2 色彩策略

当前深色基调是成立的，但蓝色层次有些太平均了。

建议：

- 页面主基底继续保留深蓝夜色
- 外部内容使用冷蓝强调
- 内部内容使用低饱和琥珀 / 金棕强调
- 飞书登录使用品牌色，仅出现在登录相关组件

这样颜色会承担“信息分层”的职责，而不只是装饰。

### 5.3 组件形态

当前很多元素都带：

- 边框
- 深色底
- 圆角

单独看都没问题，但全站叠加后会显得有点“每个东西都在发光”。

建议收一收：

- 重要卡片保留轮廓
- 次级控件减少边框存在感
- 把“区块层级”更多交给 spacing 和背景明度差，而不是全靠描边

### 5.4 排版策略

正文区的阅读感已经不错，但首页列表和头部还可以更像“出版物”而不是“后台列表”。

建议：

- 首页标题层级更明显
- 副标题更短、更像定位句
- 列表摘要控制在 1 到 2 行
- metadata 更轻，让标题和摘要成为主角

## 6. 我建议你下一轮具体改什么

### 方案 A：低风险、两天内可落地

适合先做一轮气质统一，不大改结构。

包含：

- 压缩头部高度
- 重新组织登录状态和主题切换
- 列表增加摘要
- 内部 / 外部标记重新设计
- 阅读页工具栏降噪
- 关于区文案重写

收益：

- 开发成本低
- 很快能看到整体质感提升

### 方案 B：中等改造，做成真正的知识首页

适合你把这个站长期当成“公开内容 + 内部工作知识”双模式站点。

包含：

- 首页增加 `全部 / 外部 / 内部` 模式切换
- 列表改成卡片浏览
- 筛选区改成更轻的辅助模块
- 登录态变成模式切换的一部分
- 详情页工具动作二级化

收益：

- 站点定位更清楚
- 更适合内容规模继续增长

## 7. 我最推荐的方案

如果是我来带这一版，我会选：

**先做方案 A，再用方案 A 的视觉语言承接方案 B。**

原因很简单：

- 现在最大问题不是“功能不够”，而是“角色太多，缺少主次”
- 先把视觉层级、登录状态、首页列表整理清楚，整站就会立刻更顺
- 等这层顺了，再升级成模式化知识首页，风险会小很多

## 8. Sally 的结论

huda，这个站现在最有价值的地方，不是它已经像一个成熟 CMS，而是它已经长出了一个非常鲜明的雏形：

- 它有个人表达的温度
- 有内部知识站的秩序
- 还有一点内容生产工具的能力

你现在要做的不是继续“加功能”，而是帮这三个角色排座位。

谁坐主位，谁做辅助，谁在特定场景才出现。

一旦这个秩序建立起来，站点会一下子从“能用的系统”变成“有气质的产品”。
