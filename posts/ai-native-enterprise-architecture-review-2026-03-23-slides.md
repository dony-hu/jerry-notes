# 企业级 SOP 场景，为什么不能停留在玩具模式
@tag: 困境
@part: P2/10
- 企业不会为“能跑通”买单，只会为“能生产”买单
- 单靠 AI：输出不稳定、难复现、不可审计
- 单靠 RPA：适应性弱、维护成本高、对环境变化敏感
- 真正的问题不是有没有 AI，而是能否形成生产级能力
@emphasis: AI 负责理解，自动化负责执行，平台负责治理。

---

# 在确定性场景里，自动化工具比人更稳、更快、更可审计
@tag: 自动化工具
@part: P3/10
- 适合规则明确、流程稳定、输入输出可定义的企业场景
- AI 负责理解任务，Workflow 负责编排流程，浏览器自动化负责执行动作
- 每一步都可记录、可回放、可解释
- 高重复劳动交给机器，边际成本持续下降
- 标准化后可统一调度、统一监控、统一治理

---

# Skill 脚本：把 AI 的不确定性收敛为企业可控性
@tag: Skill
@part: P4/10
@subtitle: 过程可追溯 · 结果可重复 · 失败可重试
- Skill 脚本不是单一脚本，而是连接输入、场景和执行动作的中间层
- 它把复杂任务拆成可执行、可回放、可治理的步骤
- 传统自动化：一次失败就中断，重跑容易产生脏数据
- Skill 机制：自动校验、自动重试、自动跳过已完成步骤
- 它是把 AI 能力封装成企业生产能力的关键单元

---

# 企业级 AI 落地，第一前提是隔离
@tag: 安全架构
@part: P5/10
@subtitle: 设备隔离 + 职责隔离 + 权限隔离
- 设备隔离：AI 运行在独立环境中，不与员工日常办公环境混用
- 职责隔离：一个 Agent / Skill 只负责一类任务，不让单点能力承担全部职责
- 权限隔离：AI 只能访问被授权的数据、工具和流程，不能越权操作
- 隔离的目的不是增加复杂度，而是让企业在可控前提下真正把 AI 用起来

# 实战案例：1688 询价流程如何被 Skill 自动化
@tag: 案例
@part: P6/10
- 把原本依赖人工反复执行的询价流程，封装成可重复、可复用、可扩展的业务 Skill
- 用户提出需求 → AI 理解任务 → 浏览器自动化执行 → 输出结构化结果
- 这类高频、规则明确、重复劳动重的场景，最适合优先自动化
- 案例价值不在“演示能跑”，而在“流程能沉淀、结果能复用、能力能复制”

# AI 时代，我们卖什么？
@tag: 产品定义
@part: P7/10
@subtitle: 我们卖的不是单点工具，而是可持续进化的数字生产力
@emphasis: Agent + Memory + Skill = 未来的生产力
- Agent：让软件从“被使用”变成“主动工作”
- Memory：让经验和知识不再随着人流失
- Skill：让优秀做法被复制、组合和放大
- 软件不再只是功能集合，而是新的生产力系统

# 企业的核心，不是系统，而是业务与资产
@tag: 业务价值
@part: P8/10
- 资产不只是钱和系统，还包括客户关系、品牌、知识和组织能力
- 所有技术投入，最终都应服务业务价值和资产沉淀
- 我们要配合业务打仗，而不是配合 IT 打仗
- AI、流程、平台都只是手段，长期可积累的业务资产才是目标

# 最后的结论
@tag: 收尾
@part: P10/10
- 企业要的不是“看起来很聪明的 AI”，而是“敢放进业务里的 AI”
- 从自动化工具到 Skill，再到隔离架构，本质上都在解决一件事：把 AI 变成生产力
- 真正的门槛，不在单个模型能力，而在能否把 AI、自动化和企业平台做成一套可治理、可复制、可持续优化的体系
- 下一阶段的竞争，不是谁先喊 AI，而是谁先把 AI 用成真实业务能力
