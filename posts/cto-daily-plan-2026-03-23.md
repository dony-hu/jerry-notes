## 今日工作计划

### 泛政一部

#### 吴江场景验证线（何明）
1. 以当前已跑通的一键部署系统和 Story 结果为基础，本周重点与客户确认实际效果。
2. 在客户反馈基础上，梳理下一步优化方向，形成问题清单与迭代建议。
3. 从今天开始，何明会拿这套系统到华东其他项目试用，优先结合一线交流验证可迁移性。
4. 今日重点场景：前往**无锡**与客户交流，验证系统效果和适配情况。

#### 部署互联网化能力线（刘志远统筹）
1. 在现有一键部署能力基础上，继续把整套部署能力进一步互联网化。
2. 将部署能力发布到互联网上，使后续用户可以通过互联网完成一键部署。
3. 输出互联网发布方案、部署路径、基础运维要求与风险边界。

#### 离线版本标准化线（刘志远）
1. 本周推进离线版本标准化，目标是将离线版本变成标准化、可部署的软件。
2. 选择若干区域地图地址数据，准备发布到互联网上，支撑标准化部署和试用。
3. 明确离线包与在线部署的边界、适用场景与维护方式。

#### G1内部试用推广线（向浩）
1. 本周在 G1（泛政一部）内部尽可能多地推广试用。
2. 覆盖场景包括：南山、深圳公安、嘉兴，以及后续需要参与的解决方案场景。
3. 当前阶段定位为产品内测，目标是尽快在内部真实使用起来。
4. 为下周公测做准备。
5. 公测目标是把产品推送到客户界面，让客户直接看到效果。

#### 市场支撑体系建设线（向浩）
1. 本周启动面向销售、售前、交付的一套市场支撑体系建设。
2. 基于当前产品能力和产品背后的资料、方案、知识沉淀构建支撑体系。
3. 形成一个核心智能体和一组小助手，作为市场支撑入口。
4. 让销售、售前、交付人员通过这些助手连接到整套支撑体系。
5. 一方面持续回流客户需求、问题、反馈；另一方面持续输出最新产品支持方案与产品迭代信息。
6. 形成“客户需求回流—产品支持输出—产品持续迭代”的闭环。

### 基础产品 / 平台运营

#### 后台管控与运营中心建设线（杨晓平负责，刘志远统筹）
1. 本周启动后台管控运营中心建设。
2. 目标是形成一个集中化的管控运营中心，能够对接各个产品部署实例。
3. 将各实例的使用情况、反馈情况、运行情况统一抓回，形成持续迭代的数据基础。
4. 通过“部署实例—使用反馈—产品迭代”闭环，推动产品进入持续运营阶段。

### 企业业务部 / 市场部门

#### 生态会筹备线（市场部门负责，产品组支撑）
1. 本周持续推进生态会准备。
2. 市场部门负责整体筹备与对外组织。
3. 产品组负责支撑：产品战略表达、当前进展说明、下一步规划输出、Demo 与材料支撑。
4. 统一对外表达口径，确保战略、产品、进展、规划四部分一致。
